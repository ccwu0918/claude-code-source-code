# 遙測與隱私分析

> 基於 Claude Code v2.1.88 反編譯原始碼分析

## 概述

Claude Code 實現了雙層分析管道，收集大量環境和使用者中繼資料。雖然沒有證據顯示存在鍵盤記錄或原始碼竊取，但收集範圍之廣和無法完全退出的情況引發了合理的隱私擔憂。

## 資料管道架構

### 第一方日誌 (1P)

- **端點**: `https://api.anthropic.com/api/event_logging/batch`
- **通訊協定**: OpenTelemetry + Protocol Buffers
- **批次大小**: 每批最多 200 個事件，每 10 秒重新整理一次
- **重試機制**: 二次方退避，最多 8 次嘗試，失敗事件持久化到磁碟
- **儲存位置**: `~/.claude/telemetry/`

來源: `src/services/analytics/firstPartyEventLoggingExporter.ts`

### 第三方日誌 (Datadog)

- **端點**: `https://http-intake.logs.us5.datadoghq.com/api/v2/logs`
- **範圍**: 僅限 64 種預先核准事件類型
- **Token**: `pubbbf48e6d78dae54bceaa4acf463299bf`

來源: `src/services/analytics/datadog.ts`

## 收集了什麼

### 環境指紋

每個事件都攜帶以下中繼資料 (`src/services/analytics/metadata.ts:417-452`)：

```
- platform, platformRaw, arch, nodeVersion
- 終端機類型
- 已安裝的套件管理員和執行環境
- CI/CD 檢測、GitHub Actions 中繼資料
- WSL 版本、Linux 發行版、核心版本
- 版本控制系統類型
- Claude Code 版本和建置時間
- 部署環境
```

### 程序指標 (`metadata.ts:457-467`)

```
- 執行時間、rss、heapTotal、heapUsed
- CPU 使用率和百分比
- 記憶體佔用詳細資訊
```

### 使用者追蹤 (`metadata.ts:472-496`)

```
- 正在使用的模型
- 工作階段 ID、使用者 ID、裝置 ID
- 帳戶 UUID、組織 UUID
- 訂閱等級 (max, pro, enterprise, team)
- 儲存庫遠端 URL 雜湊 (SHA256 前 16 位元)
- 代理類型、團隊名稱、父工作階段 ID
```

### 工具輸入日誌

預設截斷工具輸入：

```
- 字串: 512 個字元處截斷，顯示 128 + 省略號
- JSON: 限制 4,096 個字元
- 陣列: 最多 20 項
- 巢狀物件: 最多 2 層
```

然而，當設定 `OTEL_LOG_TOOL_DETAILS=1` 時，**完整工具輸入會被記錄**。

### 檔案副檔名追蹤

涉及 `rm, mv, cp, touch, mkdir, chmod, chown, cat, head, tail, sort, stat, diff, wc, grep, rg, sed` 的 Bash 指令，其檔案參數的副檔名會被擷取並記錄。

## 無法退出的問題

第一方日誌管道**無法被關閉**（對於直接使用 Anthropic API 的使用者）。

`isAnalyticsDisabled()` 僅在以下情況返回 true：
- 測試環境
- 第三方雲端服務提供者 (Bedrock, Vertex)
- 全域遙測退出（設定介面未公開此選項）

**沒有面向使用者的設定可以停用第一方事件日誌。**

## GrowthBook A/B 測試

使用者在不知情的情況下被分配到實驗組。系統傳送的使用者屬性包括：

```
- id, sessionId, deviceID
- platform, organizationUUID, subscriptionType
```

## 關鍵結論

1. **資料量**: 每個工作階段收集數百個事件
2. **無法退出**: 直接 API 使用者無法停用第一方日誌
3. **持久化**: 失敗事件儲存到磁碟並積極重試
4. **第三方共享**: 資料傳送到 Datadog
5. **工具詳細資訊後門**: `OTEL_LOG_TOOL_DETAILS=1` 啟用完整輸入記錄
6. **儲存庫指紋**: 儲存庫 URL 被雜湊後傳送用於伺服器端關聯
