# 遠端控制與緊急開關

> 基於 Claude Code v2.1.88 反編譯原始碼分析

## 概述

Claude Code 實現了多種遠端控制機制，允許 Anthropic（和企業管理員）在無需使用者明確同意的情況下修改行為。

## 1. 遠端託管設定

### 架構

每個符合條件的工作階段從以下端點獲取設定：
```
GET /api/claude_code/settings
```

來源: `src/services/remoteManagedSettings/index.ts`

### 輪詢行為

```typescript
const POLLING_INTERVAL_MS = 60 * 60 * 1000 // 每小時
const DEFAULT_MAX_RETRIES = 5
```

每小時靜默輪詢一次，最多 5 次重試。

### 資格

- Console 使用者 (API key): 全部符合
- OAuth 使用者: 僅 Enterprise/C4E 和 Team 訂閱者

### "接受否則退出" 對話框

當遠端設定包含"危險"變更時，會顯示阻斷對話框：

```typescript
// src/services/remoteManagedSettings/securityCheck.tsx:67-73
export function handleSecurityCheckResult(result: SecurityCheckResult): boolean {
  if (result === 'rejected') {
    gracefulShutdownSync(1)  // 退出碼 1，直接終止
    return false
  }
  return true
}
```

拒絕遠端設定的使用者，**程式直接退出**。使用者只有兩個選擇：接受遠端設定，或者 Claude Code 關閉。

### 故障容錯

遠端伺服器無法連線時，使用快取的舊設定。一旦設定過，就**永遠無法完全擺脫**遠端控制。

## 2. Feature Flag 緊急開關

多種功能可以透過 GrowthBook feature flag 遠端停用：

### 繞過權限 Kill Switch

```typescript
// src/utils/permissions/bypassPermissionsKillswitch.ts
// 透過 Statsig gate 停用繞過權限功能
```

### 自動模式斷路器

```typescript
// src/utils/permissions/autoModeState.ts
// autoModeCircuitBroken 狀態阻止重新進入自動模式
```

### 快速模式 Kill Switch

```typescript
// src/utils/fastMode.ts
// 從 /api/claude_code_penguin_mode 取得狀態
// 可以永久停用使用者的快速模式
```

### 分析 Sink Kill Switch

```typescript
// src/services/analytics/sinkKillswitch.ts:4
const SINK_KILLSWITCH_CONFIG_NAME = 'tengu_frond_boric'
```

### 語音模式 Kill Switch

```typescript
// src/voice/voiceModeEnabled.ts:21
// 'tengu_amber_quartz_disabled' — 語音模式緊急關閉
```

## 3. 模型覆蓋系統

Anthropic 可以遠端覆蓋內部員工使用的模型：

```typescript
// src/utils/model/antModels.ts:32-33
// @[MODEL LAUNCH]: Update tengu_ant_model_override with new ant-only models
```

`tengu_ant_model_override` GrowthBook flag 可以：
- 設定預設模型
- 設定預設 effort level
- 追加系統提示詞
- 定義自訂模型別名

## 總結

| 機制 | 範圍 | 使用者同意 |
|------|------|---------|
| 遠端託管設定 | Enterprise/Team | 接受或退出 |
| GrowthBook feature flags | 所有使用者 | 無 |
| Kill switches | 所有使用者 | 無 |
| 模型覆蓋 | 內部 (ant) | 無 |
| 快速模式控制 | 所有使用者 | 無 |

遠端控制基礎設施極其廣泛，且在很大程度上沒有使用者可見性或同意機制。企業管理員可以強制執行使用者無法覆寫的原則，Anthropic 可以透過 feature flag 遠端更改任何使用者的行為。
